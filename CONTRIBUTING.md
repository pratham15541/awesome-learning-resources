# Contributing Guide

Thank you for helping improve this quantum computing resource.

We welcome contributions for:

- Code examples
- Jupyter notebooks and tutorials
- Book/resource recommendations
- Bug fixes and roadmap improvements

## What You Can Contribute

## 1) Code

Examples:

- Qiskit circuits
- Algorithm implementations (Grover, QFT, etc.)
- Practice exercises and solutions

Guidelines:

- Keep examples beginner-friendly when possible
- Include clear comments for non-obvious logic
- Add a short usage section in the file or PR description

## 2) Tutorials and Notebooks

Examples:

- Step-by-step notebooks for each phase
- Visual explanations of superposition/entanglement
- Project walkthroughs

Guidelines:

- Start each tutorial with prerequisites and learning goals
- Prefer small, focused notebooks over very large ones
- Include expected output or interpretation notes

## 3) Books and Learning Resources

Examples:

- New book suggestions
- Free course links
- Talks, articles, and official documentation

Guidelines:

- Prefer high-quality and beginner-accessible resources
- Include a 1-2 sentence reason why the resource is useful
- Avoid copyrighted uploads unless redistribution is allowed

## Before You Submit

1. Check if a similar PR already exists.
2. Keep your changes focused on one topic.
3. Update README if your contribution changes the roadmap.

## Pull Request Process

1. Fork the repository.
2. Create a branch with a descriptive name.
3. Commit changes with clear commit messages.
4. Open a pull request with:
   - What changed
   - Why it helps learners
   - Any screenshots/output if relevant

## Style and Quality

- Use clear, simple language
- Prefer practical examples over heavy theory in beginner sections
- Keep file names descriptive and consistent

## Code of Conduct

- Be respectful and constructive
- Give actionable feedback
- Support beginners and contributors at all levels

## Need Help?

If you are not sure where to start, open an issue with:

- Your idea
- Target audience (beginner/intermediate)
- Expected output (code, notebook, resource list)

We can help shape it into a good first contribution.
